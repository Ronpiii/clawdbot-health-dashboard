## nightly build #34: arc pace (00:00 UTC)

### what
- built `arc pace` — work rhythm analyzer
- uses git commit timestamps to reveal natural work patterns
- session detection (50-min gap threshold), flow state classification, peak hour clustering
- first run (30 days): 246 commits, 91 sessions, 63.7h total

### sections
- **hourly heatmap**: 24-hour activity density with peak highlighting (cyan)
- **time blocks**: early morning / morning / afternoon / evening / night split with percentages
- **day-of-week**: per-day hours + deep session count
- **session types**: flow (3h+, 6+ commits), deep (2h+), sprint (<2h intense), scattered
- **metrics**: avg session length, deep work ratio, flow ratio, context switch rate, break patterns
- **recommendations**: actionable scheduling advice based on patterns

### first run findings
- peak hours: 14:00-17:00 tallinn (afternoon cluster)
- strongest day: monday (3 deep sessions, 17h44m total)
- time split: night 37%, afternoon 34%, evening 22%, morning 6%
- session types: 60% scattered, 22% deep, 11% flow, 10% sprint
- deep work ratio: 22% (low — many scattered commits from automated/maintenance work)
- longest flow: 3.5h with 32 commits (anivia + clawd sprint)
- avg break: 4h15m (long gaps = overnight/between-session breaks, expected)
- context switch rate: 0.2/session (low — work is project-focused, not scattered)

### recommendations generated
- protect 14:00-17:00 for deep work (peak productive window)
- schedule meetings at 09:00 (lowest-energy coding hour)
- monday best day for hard work (3 deep sessions historically)
- 22% deep work ratio flagged — too many short sessions

### design decisions
- contiguous peak-hour clustering: raw top-4 peak hours could span 02:00-16:00 (useless). clustering algorithm finds hours within 3h of each other and picks the densest group → actionable "14:00-17:00" recommendation
- tallinn timezone adjustment (UTC+2) applied to all hour/day calculations
- break analysis: only counts breaks < 14 hours (ignores overnight gaps)
- session types are exclusive hierarchy: flow > deep > sprint > scattered
- commit rate tracked per session (commits/hour) but not displayed in main view (available in --json)

### commands
- `arc pace` — full dashboard
- `arc pace --short` — one-liner
- `arc pace --sessions` — detailed session list
- `arc pace --recommendations` — all scheduling advice
- `arc pace --week` — this week only
- `arc pace --days 90` — custom range
- `arc pace --json` — machine-readable
