# 2026-02-18

## nightly build: color palette auditor (00:00 UTC)

### what
- built `arc colors` — color palette auditor and generator
- scans all project globals.css files, extracts colors (oklch/hsl/hex), validates WCAG contrast
- generates monochrome-first palettes with optional accent colors

### features
- oklch→sRGB color math (conversion, luminance, contrast ratio)
- achromatic vs chromatic split with gray scale ramp visualization
- terminal 24-bit ANSI color swatches
- contrast pair validation against WCAG AA/AAA standards
- auto-detects additional contrast pairs from naming patterns
- monochrome score (% achromatic non-chart colors)
- palette health score (weighted: 40% monochrome + 60% contrast)
- palette generator: monochrome-first light+dark mode, 14 named accent hues
- copy-paste CSS output in oklch format

### first run
- anivia: 89/100 (94% mono, 77 colors, 1 sidebar-accent contrast fail)
- tuner: 88/100 (89% mono, 38 colors, muted text AA-lg only)
- collabo-v2: 80/100 (50% mono, but 100% contrast on detected pairs)
- ventok-site: 28/100 (custom naming = fewer detected pairs)

### context
- directly follows ron's design research deep dive (2026-02-17)
- ron strongly prefers monochrome-first, hated pencil.dev's AI colors
- this tool enforces those design principles with measurable scores
- generates compliant palettes on demand (`arc colors --generate --accent blue`)

### commands
- `arc colors` — audit all projects
- `arc colors anivia` — audit specific project
- `arc colors --generate` — pure monochrome palette
- `arc colors --generate --accent blue` — monochrome + accent
- `arc colors --short` — one-liner per project
- `arc colors --json` — machine-readable
- aliases: `arc colors`, `arc palette`
